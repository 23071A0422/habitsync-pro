HabitSync: Social Habit Tracker
HabitSync is a full-stack habit-tracking application built to help users build consistency through social accountability. Users can track personal habits, view progress analytics, compete on leaderboards, and follow friends to stay motivated.

Features
Full Authentication: Secure signup and login powered by Supabase Auth.

Habit Dashboard: Create, edit, and track daily habits with real-time streak updates.

Social Feed: A dedicated "Friends Activity" feed showing recent check-ins and new journeys.

Dual Leaderboards: Toggle between Global and Friends-only rankings based on wellness points.

AI Habit Coach: Personalized coaching advice generated by the Gemini AI model.

Advanced Profile Management: Custom avatar cropping, notification toggles, and account reset/deletion options.

Tech Stack
Frontend: React (Vite), Tailwind CSS, Headless UI.

Backend/Database: Supabase (PostgreSQL).

AI Integration: Google Gemini API.

State Management: React Hooks (useState, useEffect).

Environment Variables
To run this project locally, create a .env file in the root directory and add the following:

Code snippet

VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
VITE_GEMINI_API_KEY=your_google_gemini_api_key

Database Setup
The application requires the following tables in your Supabase PostgreSQL database:

profiles: Stores user identity and wellness points.

habits: Stores habit configurations and current streaks.

habit_logs: Records every check-in and "New Habit" event.

follows: Manages follower/following relationships.

Required SQL Constraints
To ensure the social feed and deletion logic work correctly, run these commands in the Supabase SQL Editor:

SQL

-- Ensure data wipes when a habit or user is deleted
ALTER TABLE habit_logs ADD CONSTRAINT habit_logs_habit_id_fkey 
FOREIGN KEY (habit_id) REFERENCES habits(id) ON DELETE CASCADE;

-- Refresh relationship cache
NOTIFY pgrst, 'reload schema';
